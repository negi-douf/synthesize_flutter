# Flutter

## 概要
- 開発元はGoogle
- 2018にファーストリリース
- オープンソースのUI SDK
- 実はモバイルだけではない
	- Web, Windows, macOS, Linux

## 特徴
- 独自の描画エンジン
	- Impeller
	- Skia
	- ネイティブコンポーネントに依存しない
	- Flutter自身が直接描画
	- 60-120fpsと高速？
- ホットリロード
- 豊富なウィジェット
	- Flutterではすべての要素がウィジェットで管理される
	- iOS風、Android風のデザインを再現できる
	- オリジナルも

## 生まれた背景
- 異なるプラットフォームへの対応の手間
- 当時のWebベースのクロスプラットフォーム技術が遅かった
	- Apache Codova
	- React Native
- OS更新で崩れやすい
- など

## 歴史
- 2015 Skyとしてコンセプト発表
- 2018.12に 1.0リリース

## 言語はDart
- 歴史
	- ひとことで: JavaScriptの代替を狙って失敗したが、Flutterによって大復活を遂げた言語
	- 2011年、Chromeの開発陣によって発表
	- ChromeにDartを動かすエンジンを載せるほどの計画
	- しかし業界のブラウザたちから反対される
		- 特定の企業の言語をブラウザの標準にはしない
	- 結局、JSにトランスパイルして動かすしかなく、存在意義が薄れる
	- 2015年にはGoogleの失敗作といわれるまでに失墜
	- Flutterによる復活
		- 同じくGoogle内で新しいモバイルUIフレームワークを開発していたチームが目をつける
		- 特徴がマッチ
			- 効率的な開発
			- 高速な動作
- 特徴
	- 開発時は JITコンパイル
		- 変更をすぐに確認できるのがモバイルアプリと好相性
	- 本番時はAOTコンパイル
		- Ahead of Time
		- すべて機械語に翻訳
		- 高速性のため
	- C#, JS, TSに近いフレンドリーな文法
	- 静的型付け
	- オブジェクト指向
