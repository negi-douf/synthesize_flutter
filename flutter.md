# Flutter

## 概要
- 開発元はGoogle
- 2018にファーストリリース
- オープンソースのUI SDK
- 実はモバイルだけではない
	- Web, Windows, macOS, Linux

## 特徴
- 独自の描画エンジン
	- Impeller
	- Skia
	- ネイティブコンポーネントに依存しない
	- Flutter自身が直接描画
	- 60-120fpsと高速
		- 再描画の高速性
			- UIの必要なところしか変えない
				- 要素のツリーを持っていて、種類とキーが変わっているときだけ
				- ピクセル単位で再描画
			- ウィジェットの再生成も軽量？
		- 60-120fpsは一般的に見てもかなりなめらか
		- フレームごとの処理が軽く、フレーム落ちしないということ
- ホットリロード
- 豊富なウィジェット
	- Flutterではすべての要素がウィジェットで管理される
		- "Everything is a widget"
		- ボタン・テキストといった要素も、配置ルールもウィジェット
		- 状態・機能もウィジェット
		- なぜ？
			- 要素の組み立てと再利用が楽
				- 従来一般的だった継承という概念を捨てた
					- Viewを継承して TextViewみたいな
					- 文字と画像とアニメーションが入るボタン、とかだと問題がある
						- どれを継承する？
						- 余計なものまで継承に入って肥大化
							- バグの原因に
				- 合成 (Composition) という設計思想
					- 単一の小さな責任を持つ小さなウィジェットを結合して要素を作る
					- 小さな責任の例
						- 余白だけ
						- 中央寄せだけ
						- サイズ固定だけ
						- 塗りだけ
	- iOS風、Android風のデザインを再現できる
		- Cupertino
			- iOSそっくりの見た目と動きを再現するウィジェット集
			- Apple本社の所在地に由来する
		- OSごとに自動で切り替える分岐を書く
		- もちろん同じ独自のデザインで統一することもできる
	- オリジナルも

## 生まれた背景
- 異なるプラットフォームへの対応の手間
- 当時のWebベースのクロスプラットフォーム技術が遅かった
	- Apache Codova
	- React Native
	- PhoneGap
	- ブラウザの描画が遅い
		- HTML, CSS, JS 3つの異なる要素を読み込み
		- ピクセル変換を毎フレームやる
		- 処理のステップ
			- 1_ HTML, CSS解析
			- 2_ DOMツリー構築
				- JSで要素を動かすたびに再計算するので重い
			- 3_ 座標計算
				- 一部が変わっただけで全体を再計算することが多く、重い
			- 4_ 色塗り
			- 5_ 合成
	- JSの言語特性による遅さ
		- インタプリタのオーバーヘッド
		- V8といったブラウザのエンジンが優秀でもネイティブには及ばない
		- シングルスレッド
			- 基本的に描画もユーザー操作もメインスレッド上でおこなわれる
	- ブラウザベースのアプリは OS機能に直接アクセスできない
		- 1_ JSコード
		- 2_ ブリッジ
		- 3_ OSネイティブ
- OS更新で崩れやすい
- など

## 歴史
- 2015 Skyとしてコンセプト発表
- 2018.12に 1.0リリース

## 言語はDart
- 歴史
	- ひとことで: JavaScriptの代替を狙って失敗したが、Flutterによって大復活を遂げた言語
	- 2011年、Chromeの開発陣によって発表
	- ChromeにDartを動かすエンジンを載せるほどの計画
	- しかし業界のブラウザたちから反対される
		- 特定の企業の言語をブラウザの標準にはしない
	- 結局、JSにトランスパイルして動かすしかなく、存在意義が薄れる
	- 2015年にはGoogleの失敗作といわれるまでに失墜
	- Flutterによる復活
		- 同じくGoogle内で新しいモバイルUIフレームワークを開発していたチームが目をつける
		- 特徴がマッチ
			- 効率的な開発
			- 高速な動作
- 特徴
	- 開発時は JITコンパイル
		- 変更をすぐに確認できるのがモバイルアプリと好相性
	- 本番時はAOTコンパイル
		- Ahead of Time
		- すべて機械語に翻訳
		- 高速性のため
	- C#, JS, TSに近いフレンドリーな文法
	- 静的型付け
	- オブジェクト指向
